# Sentinel Control Product Spec

## Product Lock

Product name: **Sentinel Control**

First commercial wedge: **Sentinel GTM Operator**

Platform moat: **AgentOps Firewall**

Primary ICP v1:

- Founders validating a new product idea.
- Indie builders looking for a first-customer path.
- Agencies and startup studios turning market signals into client-ready GTM assets.

Primary output v1: **First Customer / GTM Pack**

Sentinel Control is not a generic assistant. It is a proof-backed business decision and execution agent. Every recommendation must connect to evidence, risk, approval, and traceability.

## Core Workflow V1

1. User enters an idea.
2. CueIdea validates demand in Sprint 2.
3. Research and competitor context enrich the idea in later sprints.
4. Decision agents produce build / pivot / niche down / kill / research more.
5. GTM Operator generates a pack with ICP, positioning, landing copy, outreach drafts, interview script, prospect sources, and a 7-day plan.
6. AgentOps Firewall reviews every proposed action.
7. User approves actions that require approval.
8. Safe executors create folders, markdown files, drafts, exports, and trace logs.

## V1 Non-Goals

- No autonomous email sending.
- No unrestricted desktop control.
- No browser form submission.
- No auto-code modification.
- No skill marketplace.
- No shell execution.
- No hidden evidence or hidden action execution.

## Product Rule

Every feature must map to one of:

1. Sentinel GTM Operator.
2. AgentOps Firewall.

Generic chatbot features are out of scope.
