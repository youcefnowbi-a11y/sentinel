# Cloak Browser Power Review

## Source

External source:

```text
https://github.com/CloakHQ/CloakBrowser
```

Observed purpose: stealth Chromium / Playwright-compatible browser tooling with
fingerprint patches, session/profile behavior, humanized input options, proxy
and locale/timezone alignment, and detection-oriented claims.

## Doctrine

CloakBrowser powers are harvested as high-power browser capabilities.

They are not discarded. The Brain decides whether to use them based on mission
authority, legality, risk, compliance, domain policy, evidence, expected value,
receipts, and FinalGate.

```text
Sentinel does not become powerful by deleting dangerous capabilities.
Sentinel becomes powerful by controlling when, why, where, and how capabilities
are used.
```

## Controlled Power Classification

```text
P0 Normal browser reliability
- stable Playwright-compatible launch
- session continuity
- viewport/locale/timezone consistency
- profile management

P1 Human-like operation
- mouse movement
- scroll behavior
- keyboard cadence
- interaction realism

P2 Fingerprint consistency
- browser fingerprint stabilization
- WebRTC/IP leak prevention
- timezone/proxy/locale alignment
- persistent profile realism

P3 Detection-resilience research
- bot-detection test harnesses
- browser detection diagnostics
- reliability scoring

P4 Special-authority stealth operation
- only for explicitly authorized, lawful, domain-scoped missions
- requires compliance gate, trace, receipts, kill switch, and FinalGate

P5 Forbidden misuse objective
- fraud
- fake identity
- KYC bypass
- credential theft
- spam
- unauthorized scraping
- evasion of access controls outside lawful authority
```

P5 is not capability rejection. P5 is misuse-objective rejection. The capability
remains studied and controlled. The Brain may not use it for unlawful or
out-of-authority objectives.

## Brain Decision Rule

```text
The Brain is the power governor.
Cloak-like powers are available only through Sentinel authority evaluation.
The Brain can choose not to use a power, use a lower level, request authority
extension, or block the mission.
```

Formula:

```text
use_power =
  root_authority_allows
  AND legal_compliance_allows
  AND domain_policy_allows
  AND evidence_supports_need
  AND expected_value > risk_adjusted_cost
  AND receipts/finalgate_available
```

If a condition fails:

```text
do not execute
produce safer alternative or AuthorityExtensionProposal
```
