# Comparative Architecture Analysis

## System Comparison

| System | Strongest lesson | Main weakness | Sentinel synthesis |
| --- | --- | --- | --- |
| OpenClaw | Many tools can be unified behind one runtime gateway | Gateway can concentrate too much blast radius | Keep action-kernel idea; split every organ through authority and receipts |
| Hermes | Memory, skill indexes, hooks, and delegation make agents useful over time | Memory/skills can become hidden policy | Memory is context only; skills are scanned procedure candidates |
| OpenJarvis | Routing by cost, local/cloud fit, and telemetry is architecture | Routing can accelerate unsafe execution | Cost is a firewall dimension, not billing metadata |
| JARVIS | Sidecar and approval lifecycle can unlock machine operation | Host-level control is critical risk | Sidecar is a product surface with deny-by-default contracts |
| Financial Services repo | Domain skill packs can encode high-value professional workflows | Finance outputs need compliance and human sign-off | Finance becomes procedure/eval expertise, not automatic advice or trading |
| CloakBrowser | Browser profile realism and detection diagnostics can improve reliability | Stealth can become abuse if detached from authority | Classify all powers; Brain governs lawful, scoped, traceable use |

## Architecture Shift

Most agents are prompt-driven tool runners. Sentinel must be a mission-governed
operating layer:

```text
mission authority first
then cognition
then organ contract
then dry-run
then execution only if authorized
then receipt and FinalGate
```

## Deep Bet

The breakthrough is not one tool. The breakthrough is the stable relationship
between Brain, authority, organs, evidence, receipts, and product workflows.
