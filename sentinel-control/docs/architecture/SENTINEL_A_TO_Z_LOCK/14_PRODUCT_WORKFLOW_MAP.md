# Product Workflow Map

## GTM Operator Workflow

```text
CueIdea evidence -> Brain L4 -> GTM pack -> outreach drafts -> trace
```

Organs: local files, browser evidence later, channel drafts later.
Blocked: real send, spend, production mutation unless future authority exists.

## Capital Opportunity Workflow

```text
finance procedures -> opportunity portfolio -> risk/ROI analysis
-> optional spend authority proposal
```

Organs: finance, browser evidence, external API, future capital operator.
Blocked now: spend, trade, payment, account creation.

## Browser Evidence Workflow

```text
browser organ -> source capture -> evidence chain -> belief update -> FinalGate
```

Strong browser powers are selected by `BrowserPowerGovernor` only when lawful,
authorized, scoped, and necessary.

## API Data Acquisition Workflow

```text
external API organ -> dry-run call preview -> rate/cost/privacy check
-> data receipt
```

No mutation until API organ reaches promotion level with authority.

## Outbound Prospecting Workflow

```text
lead research -> drafts -> compliance review -> future send authority
```

Draft-first until channel organ and approval gate are locked.

## Account Setup Workflow

```text
account setup checklist -> dry-run -> approval -> scoped account ops
```

Fake identity, KYC bypass, and hidden subscriptions are misuse objectives and
must be blocked.

## Research-To-Action Workflow

```text
evidence -> belief update -> debate -> action score -> resourcefulness
-> controlled execution if authorized
```

This is the generic Mission OS loop.
